# CLAUDE.md — backend

Go-сервис мессенджера. Общие мандаты — в корневом [`../CLAUDE.md`](../CLAUDE.md).
Архитектура и API — в [`README.md`](README.md).

## Команды

```bash
go run ./cmd/server     # запуск (:8080); миграции применяются на старте (goose)
go test ./...           # тесты (интеграционные — на testcontainers, нужен Docker)
go vet ./... && gofmt -l .   # перед коммитом
```

## Архитектура — соблюдать строго

Чистая архитектура, зависимости только внутрь: `domain ← usecase ← adapter`.

- **`domain/`** — сущности и инварианты. **Без** импортов фреймворков, БД, HTTP. Чистый Go.
- **`usecase/<feature>/`** — бизнес-логика. Зависит от `domain` и **только от интерфейсов** (`ports.go`),
  не от конкретных репозиториев/адаптеров. Новую зависимость объявлять портом в `ports.go`.
- **`adapter/`** — реализация портов: `delivery/http` (chi-хендлеры), `delivery/ws`, `repo/postgres` (pgx),
  `storage/minio`, `cache/redis`, и т.д.
- **`app/`** — DI-сборка на uber/fx (`providers.go` — фабрики, `server.go` — wiring). Новый компонент
  регистрируется здесь.

Не тащи `*pgxpool`, `chi`, `redis` в usecase. Не клади бизнес-логику в хендлеры — они только парсят
запрос, зовут usecase, сериализуют ответ.

## Конвенции

- Ошибки домена — в `domain/errors.go` (`ErrNotFound`, `ErrForbidden`, `ErrTooLong`…). Хендлеры мапят
  их в HTTP-коды; не возвращай сырые ошибки наружу.
- Любой пользовательский ввод (текст, entities, ссылки) санитизировать в usecase
  (см. `usecase/chat/sanitize.go`): allow-list схем ссылок, лимиты длины/количества.
- Опциональные зависимости (Redis, MinIO, VAPID, GeoIP) деградируют мягко — код не должен падать
  при их отсутствии, фича просто отключается.

## Миграции

- Файлы: `internal/store/postgres/migrations/NNNN_name.sql` (goose), применяются **автоматически** на старте бэкенда.
- Нумерация строго последовательная; новый файл — следующий номер. Не править уже применённые миграции —
  только новые.
- jsonb-колонки: при записи через pgx передавать `string(json)`/`nil`, не `[]byte` (иначе кодируется как bytea).

## Тесты

- Интеграционные хендлер/usecase-тесты поднимают реальные Postgres/Redis/MinIO через testcontainers —
  **нужен запущенный Docker**. Часть использует `miniredis`.
- При смене сигнатуры порта обнови фейки (`*/fakes_test.go`) — иначе пакет не соберётся.

# Messenger (Telegram clone)

Полнофункциональный клон Telegram: **Go-бэкенд** (clean architecture, PostgreSQL + Redis + MinIO)
и **React/TypeScript-фронтенд** (пиксель-в-пиксель ремейк Telegram Web K) с реальной серверной
частью — REST + WebSocket, оптимистичная отправка, rich-text, медиа, истории, группы/каналы, звонки.

Это монорепозиторий: бэкенд и фронтенд лежат рядом и поднимаются единым `docker compose`.

```
messenger-denis/
├── backend/            # Go-сервис (API + WebSocket). См. backend/README.md
├── telegram-ui-clone/  # React/TS SPA. См. telegram-ui-clone/README.md
├── nginx/              # nginx.conf — раздаёт статику фронта и проксирует на backend
├── docs/               # contracts.md, ui-kit-migration.md, research/
├── client-build/       # собранная статика фронта (раздаётся nginx)
├── docker-compose.yml  # postgres + redis + minio + backend + nginx
└── .env.example        # пример переменных окружения
```

## Возможности

Клон покрывает почти всю функциональность Telegram Web K и по некоторым направлениям идёт дальше
(например, полноценные **E2E секретные чаты**, которых в оригинальном Web K нет). Ниже — подробный
разбор реализованного. Всё подтверждается кодом: REST/WS-роутами, миграциями Postgres и компонентами.

### Аутентификация и аккаунт
- **Вход по телефону + OTP** (в dev — код `12345`), **QR-логин** (вход сканированием QR со второго
  устройства, `POST /auth/qr/confirm`).
- **Активные сессии/устройства**: список, завершение отдельной сессии или всех прочих
  (`GET/DELETE /sessions`), метаданные устройства.
- **Двухэтапная аутентификация** (cloud password) и **passkeys** (WebAuthn: begin/finish).
- **Смена номера телефона** с подтверждением, **редактирование профиля** (имя, @username с проверкой
  доступности, био), **удаление аккаунта**.
- **Passcode-lock** — локальная блокировка приложения кодом.

### Сообщения
- **Rich-text**: bold/italic/underline/strike/spoiler/monospace/pre (блоки кода с подсветкой prism),
  ссылки, blockquote, `text_link`; модель `MessageEntity` с offset/length в UTF-16, разбор markdown
  на отправке, санитизация на бэке.
- **Отправка/редактирование/удаление** (для всех или у себя), **пересылка** (с указанием источника),
  **ответы с цитатой** (выделенный фрагмент), **закреплённые сообщения** (бар + отдельный экран).
- **Реакции** (эмодзи), с разбивкой и списком «кто отреагировал», бейдж непрочитанных реакций.
- **Star-реакции** — платные ⭐-реакции: списание Stars у отправителя → начисление автору,
  накопительно, топ-отправители, анонимность.
- **Эффекты сообщений** (анимация при отправке) и **платные сообщения** (плата Stars за отправку в чат).
- **Запланированные сообщения** (scheduled) с отправкой в назначенное время или «отправить сейчас».
- **Автоудаление** (TTL) — по чату и глобально для новых чатов.
- **Чек-листы** — интерактивные списки задач: пункты, кто отметил, прогресс, права на отметку/добавление.
- **Опросы и квизы** (мультивыбор, закрытие).
- **Перевод сообщений** (translate), **read-date** — «прочитано в HH:MM» в приватных чатах с
  взаимной приватностью и тумблером «Время прочтения».
- **@упоминания** с переходом к следующему непрочитанному, **треды/комментарии** к постам каналов.

### Медиа
- **Фото, видео, документы, аудио/музыка**, **голосовые** и **видеосообщения-кружки** (запись с камеры),
  **альбомы** (grouped media).
- **Чанковая загрузка с прогрессом и отменой**, серверная генерация превью/blurhash через ffmpeg,
  варианты медиа (thumbnail).
- **Медиа-редактор** (кроп, поворот, рисование, текст, стикеры на фото/видео).
- **Медиа-вьюер (lightbox)**, плеер с управлением скоростью, waveform у голосовых.
- **Paid media** — медиа, разблокируемое за Stars: до оплаты сервер не отдаёт контент
  (только blurhash/размер/цена), два слоя защиты от утечки.
- **Live-геолокация**, геоточки/venue, контакты как сообщения.

### Стикеры, GIF, эмодзи
- **Стикеры** статичные и **анимированные (Lottie)**, наборы (установка/поиск/избранное/недавние).
- **GIF** — сохранённые и поиск (Tenor-подобный).
- **Анимированные кастом-эмодзи в тексте** — рендер стикер-документа инлайн в строке (играет только
  видимый через IntersectionObserver), вкладка кастом-эмодзи в пикере.
- **Emoji-статус** у пользователя.

### Чаты, группы, каналы
- **Диалоги 1:1**, **Saved Messages** (избранное), **группы** и **супергруппы**, **каналы**.
- **Права и роли**: админы, гранулярные разрешения, баны, ограничения участников, slowmode,
  настройки реакций, видимость истории для новых, конвертация типа.
- **Приглашения**: инвайт-ссылки, заявки на вступление (approve/decline).
- **Просмотры постов** каналов, счётчики комментариев, обсуждения (discussion-группа).
- **Per-chat темы оформления** — набор тем (эмодзи + акцент + обои), общая для чата; акцент
  применяется к области чата и правой панели.
- **Статистика каналов** и **статистика поста/истории** — графики (own inline-SVG) из реальных
  данных: рост участников, просмотры по дням, топ-посты.
- **Boosts** каналов (premium-слоты, уровни) и **Giveaways** (розыгрыши premium/Stars с участием и
  розыгрышем победителей).
- **Suggested posts** — предложка постов в канал от не-админов с одобрением/отклонением админом.

### Форумы и топики
- **Форум-топики** как в Telegram: панель тем, топик = отдельная лента; per-topic непрочитанные,
  mute, mention, закрытие/скрытие/закрепление; топики ведут себя как диалоги (бейджи, галочки).

### Секретные чаты (E2E)
- **Сквозное шифрование** (WebCrypto ECDH, крипта в worker), handshake (request/accept/reject),
  верификация ключа (fingerprint), E2E-медиа.

### Истории (Stories)
- Публикация, лента историй, просмотр, список просмотревших, удаление, статистика истории.

### Звонки и трансляции
- **Аудио- и видеозвонки 1:1** (WebRTC, ICE-серверы), **групповые звонки/voice chat**.
- **RTMP-livestream** — трансляция в group call: RTMP URL + stream key (для OBS), LIVE-индикатор,
  счётчик зрителей (реального медиа-ingest нет — отдаются креды и моделируется статус).

### Профиль и контакты
- **Галерея фото профиля** (свои и чужие), аватар-кроппер.
- **Контакты**: добавление, редактирование, заметка, удаление; экран «Изменить контакт».
- **Личное фото контакта** (вижу только я) и **предложить фото** (контакт принимает → ставит себе).
- **Presence**: online/last-seen, «печатает…»/«записывает…»/«отправляет файл» индикаторы.
- **Verified/Premium** бейджи, публичные страницы `@username`.

### Приватность и безопасность
- **Правила приватности** (last-seen, время прочтения и т.п.: everybody/contacts/nobody + исключения),
  **чёрный список**, **жалобы** (report).
- Ссылки только по allow-list схем, пользовательский контент рендерится только React-нодами
  (никогда не raw HTML), лимиты на длину/число сущностей.

### Монетизация (Telegram Stars)
- **Stars**: баланс, пополнение; **подарки** (каталог, отправка, витрина в профиле, скрыть/показать,
  конвертация в Stars).
- **Telegram Premium**: экран фич, **мок-чекаут картой** (планы 1/6/12 мес), управление подпиской,
  бейдж премиума.

### Боты и mini-apps
- **Bot API** (`/bot/{token}/{method}`), создание ботов, команды, меню-кнопка, инлайн-режим,
  callback-кнопки, **WebApp/mini-apps** (модалка, cloud storage, webapp-data). Подробно — в `docs/bots/`.

### Организация
- **Папки/фильтры чатов** с инвайтами (shared folders), **архив**, **закрепление диалогов**,
  **облачные черновики** (с entities и reply), **mute** (в т.ч. временный).

### Realtime, синхронизация, поиск
- **WebSocket** `/ws?token=` с кадрами `{t,d}`: `new_message`/`edit`/`delete`/`read`/`typing`/
  `presence`/`reaction`/`poll_update`/`boost_update`/`giveaway_update`/`chat_theme_update`/
  `star_reaction`/`checklist_update`/`livestream_update`/`suggested_post_update` и др.
- **Оптимистичная отправка** (бабл сразу, затем ack/error), durable outbox (переживает перезагрузку),
  **catch-up синхронизация** (`GET /sync`, `GET /channels/{id}/difference` по pts).
- **Глобальный поиск** и **поиск по чату/сообщениям**, **Instant View** (`/iv`), **превью ссылок**
  (web page), **web-push уведомления** (VAPID).
- **Горячие клавиши** (кроссплатформенные Cmd/Ctrl/Alt), favicon-бейдж непрочитанного.

## Архитектура

```
                 ┌─────────────────────────────┐
   браузер ────► │ nginx (:8080)               │
                 │  /         → статика фронта  │
                 │  /api, /ws → backend:8080    │
                 └──────────────┬──────────────┘
                                │
                 ┌──────────────▼──────────────┐
                 │ backend (Go, chi + fx)      │
                 │  REST + WebSocket            │
                 └───┬──────────┬──────────┬───┘
                     │          │          │
              ┌──────▼───┐ ┌────▼────┐ ┌───▼─────┐
              │ Postgres │ │  Redis  │ │  MinIO  │
              │ (данные) │ │(кэш/RT/ │ │ (медиа) │
              │          │ │presence)│ │         │
              └──────────┘ └─────────┘ └─────────┘
```

- **Postgres** — основное хранилище (пользователи, чаты, сообщения, реакции, истории, контакты…).
- **Redis** — кэш сессий, QR-логин, pub/sub реалтайма, presence (online/last-seen), очередь web-push. Опционален: при недоступности сервис деградирует мягко.
- **MinIO** — объектное хранилище медиа; сервер генерирует превью через ffmpeg. Опционален: без него медиа отключается.

Подробности слоёв — в [`backend/README.md`](backend/README.md) и [`telegram-ui-clone/README.md`](telegram-ui-clone/README.md).

## Быстрый старт (Docker)

Поднять всё разом:

```bash
# 1. собрать фронтенд в client-build/ (раздаётся nginx)
cd telegram-ui-clone && npm install && npx vite build --base=/ --outDir ../client-build && cd ..

# 2. поднять стек
docker compose up -d --build
```

Сервисы и порты (из `docker-compose.yml`):

| Сервис   | Образ                  | Порт (host)   | Назначение                          |
|----------|------------------------|---------------|-------------------------------------|
| nginx    | `nginx:alpine`         | `8080` → 80   | статика фронта + прокси на backend  |
| backend  | `./backend/Dockerfile` | (внутр. 8080) | Go API + WebSocket                  |
| postgres | `postgres:16-alpine`   | `5432`        | база данных                         |
| redis    | `redis:7-alpine`       | `6379`        | кэш / realtime / presence / очередь |
| minio    | `minio/minio:latest`   | `9000`, `9001`| хранилище медиа (S3 + консоль)      |

Миграции Postgres применяются автоматически при старте бэкенда (goose).
Приложение открывается на **http://localhost:8080**.

Учётные данные по умолчанию (dev): postgres `messenger/messenger`, MinIO `minioadmin/minioadmin`,
OTP-код для входа — `12345` (`DEV_OTP_CODE`).

> Если порты 5432/6379/8080 заняты другими проектами — переопредели проброс в `docker-compose.yml`.

## Разработка

**Бэкенд** (нужны Postgres/Redis/MinIO — можно поднять только их через compose):

```bash
cp .env.example backend/.env   # или экспортировать переменные
cd backend && go run ./cmd/server
go test ./...                  # интеграционные тесты на testcontainers (нужен Docker)
```

**Фронтенд** (dev-сервера нет — watch-сборка в `client-build/`, раздаёт nginx стенда):

```bash
cd telegram-ui-clone
npm install
npm run dev      # vite build --watch → ../client-build (открывать http://localhost:38080)
npm test         # vitest
```

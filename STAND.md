# Verify-стенд: как открыть и войти

## Адрес

| Что | URL |
|---|---|
| **Приложение** | **https://localhost:38443** (сертификат самоподписанный — принять предупреждение) |
| HTTP-порт | http://localhost:38080 (только редиректит на https) |

Стек — docker-проект `msgrverify` (`docker-compose.verify.yml`), порты специально смещены,
чтобы не конфликтовать с другими проектами (5432/6379/8080 заняты).

## Вход (креды)

Вход по номеру телефона + OTP-код. На стенде код **всегда `12345`** (`DEV_OTP_CODE`).
Пользователь **создаётся автоматически** при первом входе с любым номером —
регистрация не нужна.

Уже существующие пользователи стенда (с именами и историей чатов):

| Пользователь | Телефон | Код |
|---|---|---|
| Алиса Иванова | `+7 999 000-00-01` | `12345` |
| Денис Марамыгин | `+7 999 000-00-02` | `12345` |
| Дарья Смирнова | `+7 999 000-00-03` | `12345` |
| Егор Кузнецов | `+7 999 000-00-04` | `12345` |

Для переписки — два аккаунта в разных браузерах (или обычное + инкогнито окно).
Повторный вход с тем же номером открывает тот же аккаунт (данные в Postgres стенда).

## Управление стеком

```bash
# поднять весь стенд
docker compose -p msgrverify -f docker-compose.verify.yml up -d

# пересобрать только backend после правок Go
docker compose -p msgrverify -f docker-compose.verify.yml up -d --build backend
docker compose -p msgrverify -f docker-compose.verify.yml restart nginx   # обязательно!

# фронтенд: пересборка подхватывается без рестарта контейнеров
cd telegram-ui-clone && npx vite build --base=/ --outDir ../client-build
```

## Грабли

- **nginx кэширует IP backend'а**: после пересборки backend без `restart nginx` будет 502.
- **SharedWorker**: после обновления фронта закрыть **все** вкладки стенда и открыть заново,
  иначе продолжит работать старый воркер (менеджеры живут в нём).
- Postgres стенда: `messenger`/`messenger` (внутри docker-сети, наружу не проброшен).

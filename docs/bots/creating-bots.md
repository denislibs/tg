# Создание бота

Боты создаются через системного бота **@BotFather** прямо в приложении —
1:1 как в Telegram. Реального «движка» ботов на сервере нет: вы пишете бота как
внешний сервис, а сервер лишь маршрутизирует апдейты и сообщения через
[Bot API](bot-api.md).

## Шаг 1. Создать бота

Откройте чат с **@BotFather** и отправьте:

```
/newbot
```

BotFather проведёт по шагам:

1. **Имя** — отображаемое имя бота (например `Мой магазин`).
2. **Username** — латиница, 5–32 символа, **должен заканчиваться на `bot`**
   (например `my_shop_bot`). Проверяется уникальность.

В ответ придёт **токен**:

```
Готово! Поздравляю с новым ботом. Он доступен как @my_shop_bot.

Токен для Bot API:
777123:2e3f56445f008abaed14dd816848db9f

Держите токен в секрете. Дальше: /setcommands, /newapp, /setmenubutton.
```

Токен — это `<bot_id>:<секрет>`. `bot_id` слева от двоеточия — числовой id бота.

## Команды @BotFather

| Команда | Что делает |
|---|---|
| `/newbot` | Создать нового бота (имя → username → токен) |
| `/mybots` | Список ваших ботов |
| `/token` | Показать токен бота |
| `/revoke` | Отозвать старый и выпустить новый токен |
| `/setcommands` | Задать список команд (меню «/») |
| `/newapp` | Создать mini-app (см. [Mini-Apps](mini-apps.md)) |
| `/setmenubutton` | Задать кнопку-меню mini-app у поля ввода |
| `/cancel` | Отменить текущий диалог |

### Выбор бота

Если у вас несколько ботов, команды вроде `/token`, `/setcommands`,
`/setmenubutton`, `/newapp` спросят, для какого бота. Можно сразу указать
username аргументом:

```
/setmenubutton @my_shop_bot
```

Если бот один — он выбирается автоматически.

### Задать команды

```
/setcommands
→ (выберите бота, если их несколько)
→ пришлите список, по одной команде в строке:

start - запустить
help - помощь
shop - открыть магазин
```

Эти команды покажутся в подсказке по «/» в чате с ботом.

## Шаг 2. Запустить сервис бота

Токен вставляется в ваш код. Минимальный рабочий бот — в
[`examples/bot/`](../../examples/bot/):

```bash
BOT_TOKEN=777123:2e3f56445f008abaed14dd816848db9f node examples/bot/demo-bot.mjs
```

Дальше — [Bot API](bot-api.md): как получать апдейты и отвечать.

## Замечания

- **Пользователь должен написать боту первым** (обычно `/start`) — как в Telegram,
  бот не может начать диалог сам. Технически бот шлёт сообщения только в чаты,
  участником которых он является.
- Токен даёт полный доступ к боту — не публикуйте его. При утечке используйте
  `/revoke`.
- Демо-бот `@demobot` и `@BotFather` — **внутренние** (их логика зашита на
  сервере, токена нет). Созданные вами боты — внешние, через Bot API. Они
  спокойно сосуществуют.

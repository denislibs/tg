# UI-кит / уход от MUI на современный CSS (план)

Стратегический разбор: перевести UI на CSS-переменные + современный CSS, вынести
примитивы в отдельные тонкие компоненты (стили — 1:1 из tweb), опционально —
отдельным пакетом со Storybook. Контекст: профайлинг показал, что MUI (`sx`/emotion)
— постоянный per-render налог; спайки fps лечатся отдельно (например, fold историй
уже переведён на CSS-переменную без setState на кадр).

## Факты по нашему коду
- `sx=`: ~988, `<Box>`: ~619, `<Typography>`: ~271, `useTheme(`: ~83, файлов на `@mui/material`: ~73.
- Бандл `index.js` ≈ 901 КБ (значимая доля — MUI + emotion).

## Факты по tweb
- Уже на CSS custom properties: **419 уникальных `--vars`**, смена темы через `_themes.scss`.
- Стек: Solid + vanilla TS + SCSS. → стили переносятся почти как есть, компоненты — нет.

## Что реально переносится из tweb
- ✅ Дизайн-токены (`--primary-color`, поверхности, радиусы, тени, размеры) → копируем
  `_variables.scss`/`_themes.scss` в наши CSS-vars почти дословно. Смена темы тогда —
  поставить `data-theme` на `<html>`, **ноль React-ре-рендеров**.
- ✅ Стили/классы/структура DOM компонентов (их SCSS — источник пиксель-парити).
- ❌ Сами компоненты (Solid/vanilla) — реимплемент тонкими React-обёртками вокруг той
  же разметки и классов.

## Почему это даёт перф
- Каждый `<Box sx={{...}}>` на каждый рендер: emotion сериализует объект стилей,
  читает `useTheme()`-контекст, оборачивает в `styled('div')`. Для строки чата с 11 Box
  — 11× сериализация+контекст на ре-рендер. Чистый `<div className>` со статичным CSS —
  только DOM, ноль JS-работы со стилями (так и быстрее tweb).
- Бандл/парс: выпил `@mui` + `@emotion` — меньше JS грузить/парсить/держать в памяти.
- Тема без ре-рендеров: `tg.*` (JS-объект через контекст) → `var(--tg-*)`; смена темы =
  один атрибут на `<html>`.

## Оговорки
- MUI — не причина спайков, а равномерный налог; виден, когда что-то и так часто рендерится.
- Полный выпил — переписать ~73 файла / ~988 `sx`. Большой объём и риск регрессий
  (framer-motion завязан на `Box component={motion.div}`).
- **80/20:** почти весь рантайм-выигрыш — перевод ТОЛЬКО горячих путей (ChatListItem,
  строки сообщений, хрома сайдбара) на CSS-классы. Редкие экраны (настройки, модалки)
  на практике MUI ничего не стоят.

## Решение по способу CSS: SCSS на сборке (Sass build-step)

**Выбрано: переносим `.scss` из tweb как есть, Sass работает на сборке.** Vite держит
`.scss` из коробки (`npm i -D sass`), рантайм-стоимость нулевая (компилируется в статический
CSS). Это закрывает цель плана — выпил **рантайм**-CSS-in-JS (emotion `sx`) — и при этом
сохраняет пиксель-парити с tweb: стили копируются почти дословно.

Почему не «plain CSS руками» (по факту из исходников tweb, 272 `.scss`):
- **1359 `var(--*)`** — это уже чистый CSS, переносятся как есть (бо́льшая часть токенов/тем).
- Но **663 `@include` (23 миксина), 329 `@use`, 12 `@for`, 28 `@if`/21 `@else`, 2 `@function`,
  `math.div`, `mix()`** — это build-time конструкции, у которых **нет рантайм-эквивалента** в CSS.
  Ручная конвертация = разворачивать циклы, дублировать миксины, пересчитывать функции →
  огромный объём и риск регрессий. Sass делает это сам.
- Формулировка «plain CSS рекомендуется» ниже относилась к отказу от **рантайм** CSS-in-JS,
  **не** к запрету препроцессора. Sass-at-build-time — это и есть нулевой-рантайм вариант.

Нативный современный CSS (см. ниже: `color-mix`, `@layer`, nesting, `:has`) используем для
**новых** примитивов, где он короче миксинов, и постепенно подменяем им scss-абстракции —
но не как обязательный первый шаг.

## Что нужно (чеклист)
1. **Способ CSS — решено:** SCSS на сборке (см. раздел выше). Подключить `sass` в Vite,
   `tg.*` свести к `var(--tg-*)`. Избегать рантайм-CSS-in-JS (emotion/styled).
2. **Фундамент — токены в CSS-vars:** портировать палитру/токены tweb в `:root` +
   `[data-theme="dark"]`; `tg.*` сделать алиасом на `var(--tg-*)` или заменить. Разблокирует
   всё и сразу убирает ре-рендер на смене темы.
3. **Тулинг пакета (если выносить):** workspace (pnpm/turborepo), сборка Vite lib-mode/`tsup`;
   **Storybook 8** (Vite) со светлой/тёмной темой через toolbar; линт-правило против нового
   `sx`/`@mui`.
4. **Инвентарь примитивов** (≈ по частоте рендера): `Button`/`IconButton`, `Avatar`, `Ripple`,
   `Input`/`TextField`, `Switch`/`Checkbox`, `Menu`/`Popover`, `Modal`/`Sheet`, `Tabs`, `Badge`,
   `Skeleton`; «не-компоненты» — `Box`→`div`+класс, `Typography`→текст-классы. Для каждого:
   копируем SCSS tweb → React-shell.
5. **Strangler-миграция:** по компоненту, горячие пути первыми; MUI параллельно до парити,
   затем выпил по файлам.
6. **Гварды:** ESLint `no-restricted-imports` на `@mui/*`, запрет `sx` вне legacy-зоны.

## Современный CSS, который тут зайдёт
- `@property --stories-fold` (типизированная анимируемая кастом-проперти) + scroll-driven
  animations (`animation-timeline: scroll()`) — fold историй можно сделать вообще без
  JS-скролл-обработчика.
- CSS nesting (нативно), `@layer` (каскад: tokens → base → components → utilities),
  `color-mix()` (ховеры/disabled из одного токена), logical properties (RTL бесплатно),
  `clamp()`, `:has()` (состояния без классов-модификаторов), container queries (адаптив без
  JS `useMediaQuery`), `content-visibility: auto` (дешёвый «виртуальный» список без либы).

## Про отдельный пакет
Начать в монорепо-папке `src/ui/` (с тем же Storybook), вынести в пакет позже, когда
примитивы устаканятся. Ранняя версионно-сборочная возня при одном потребителе замедлит
итерации без профита.

## С чего начать (минимальный первый шаг)
1. Подключить Sass: `npm i -D sass` (Vite подхватывает `.scss` без конфига).
2. Портировать токены/темы tweb (`_variables.scss`/`_themes.scss`) → наши CSS-vars в
   `:root`/`[data-theme]`; `tg.*` свести к алиасу на `var(--tg-*)`. Разблокирует смену темы
   без ре-рендеров.
3. Перенести 1–2 базовых partials tweb (хелперы/миксины hover, text-overflow) как есть в `.scss`.
4. Поднять Storybook (Vite) со светлой/тёмной темой.
5. 2–3 примитива на горячий путь (`IconButton`, `Avatar`, текст-классы), перевести
   `ChatListItem`, замерить react-scan до/после.

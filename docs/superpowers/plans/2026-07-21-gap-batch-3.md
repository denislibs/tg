# Батч-3 gap-фич: стикеры/GIF/анимации, медиа-редактор, IV и добивки

> **For agentic workers:** REQUIRED SUB-SKILL: subagent-driven-development. Задачи — независимые фичи,
> каждая: реализация → проверка (build/tests) → отдельный коммит на `feat/gap-features-3`.

**Цель:** закрыть основные оставшиеся пробелы против tweb (см. gap-анализ 2026-07-21).

**Решения (утверждены):**
- Стикеры/анимированные эмодзи: СВОЯ инфраструктура наборов (MTProto-ассеты недоступны), Lottie-анимации
  через lottie-web (у нас уже есть `LottieSticker.tsx`), seed открытыми наборами.
- GIF-поиск: Tenor v2 через `TENOR_API_KEY` env, бэкенд-прокси; без ключа — только «сохранённые GIF».
- Эффекты сообщений и Instant View: свои замены (canvas-анимации; Readability reader-mode).
- Референс UI/поведения — tweb 1:1 (пути указаны в задачах).

**Инварианты (все задачи):** архитектура фронта (View→hook→managers, realtimeBridge — единственный канал
socket→store, менеджеры в SharedWorker), чистая архитектура бэка (domain←usecase←adapter, порты в ports.go,
DI в app/server.go, goose-миграции с последовательными номерами от 0051), никакого raw-HTML для юзер-контента,
TS strict, тесты (usecase + testcontainers на бэке, vitest на фронте), коммит после каждой задачи.

## T1. Backend: инфраструктура стикеров и GIF

**Миграция 0051_stickers_gifs.sql:**
- `sticker_sets(id, slug unique, title, kind text 'sticker'|'emoji', created_by, created_at)`
- `stickers(id, set_id fk cascade, media_id fk media, emoji text, position int)` — media_id указывает на
  загруженный .json (lottie) или .webp/.png
- `user_sticker_sets(user_id, set_id, position, added_at, pk(user_id,set_id))` — установленные наборы
- `recent_stickers(user_id, sticker_id, used_at, pk(user_id,sticker_id))`
- `faved_stickers(user_id, sticker_id, faved_at, pk(user_id,sticker_id))`
- `saved_gifs(user_id, media_id, saved_at, pk(user_id,media_id))`

**Usecase `usecase/stickers`** (порты в ports.go, репо `adapter/repo/postgres/stickersrepo.go`):
- ListSets(userID) — установленные; SetWithStickers(setID/slug); InstallSet/UninstallSet; SearchSets(q)
- RecentStickers/FavedStickers (+Fave/Unfave, Use — LIFO лимиты 20/faved 10 как tweb)
- CreateSet(owner, title, slug, kind) + AddSticker(setID, mediaID, emoji) — для seed/админа
- SavedGifs(userID)/SaveGif/DeleteGif (лимит 200, LIFO)
- SearchStickersByEmoji(userID, emoji) — стикеры установленных наборов с этим эмодзи (для саджестов)

**HTTP** (`adapter/delivery/http/stickers_handler.go`, роуты в router.go):
- GET /sticker-sets (мои), GET /sticker-sets/{slug}, POST /sticker-sets/{id}/install, DELETE .../install
- GET /stickers/recent, /stickers/faved, POST /stickers/{id}/fave, DELETE .../fave, POST /stickers/{id}/use
- GET /stickers/search?emoji=🔥 ; GET /sticker-sets/search?q=
- POST /sticker-sets (создать; kind), POST /sticker-sets/{id}/stickers {media_id, emoji}
- GET /gifs/saved, POST /gifs/saved {media_id}, DELETE /gifs/saved/{media_id}
- GET /gifs/search?q= — прокси Tenor v2 (`https://tenor.googleapis.com/v2/search?key=…`), env `TENOR_API_KEY`
  (config + мягкая деградация: без ключа 404/empty). Ответ нормализовать: [{id, url(mp4/gif), preview, w, h}].

**Seed:** `cmd/seed-stickers/main.go` — грузит из `backend/assets/stickers/*/` (набор = папка: meta.json
{title, slug, kind, emojis[]} + файлы .json/.webp) через minio + таблицы. Добавить 2-3 открытых Lottie-набора
(например, из lottiefiles free / twemoji-цветные webp) в assets.

**Тип сообщения:** отправка стикера = обычное сообщение `type:'sticker'` с media_id стикера (медиа уже
инфраструктурно есть); WS/history уже несут медиа-мету (media_mime 'application/json' → lottie).
Убедиться, что sticker-тип проходит Send без ограничений медиа-владения: стикеры общие → в Send разрешить
media_id, принадлежащий стикеру из установленного/публичного набора (проверка через stickers usecase порт).

**Тесты:** usecase-тесты на лимиты recent/faved + testcontainers-тест репо (миграция, install/uninstall,
search by emoji, saved gifs LIFO/лимит).

## T3. Frontend: вкладка GIF + отправка

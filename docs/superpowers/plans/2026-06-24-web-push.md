# Web Push (F10) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver push notifications when a recipient is offline — a Service Worker handles `push` + `notificationclick`, and the app subscribes the browser to Web Push and registers the subscription with the backend.

**Architecture:** The backend push pipeline is already built (Plan D2: `internal/push` Notifier → Redis Stream `push:queue` → Worker → WebPushSender; `GET /push/vapid_public_key`, `POST /push/subscribe`); it is **disabled until VAPID keys are set**. This slice (1) enables VAPID in the verify stack, (2) adds a Core-Worker `PushManager` (RPC: `vapidKey`, `subscribe`), (3) adds a real **Service Worker** (`public/sw.js`) for `push`/`notificationclick`, and (4) registers the SW + runs the subscribe flow in the **window context** (the Push API and `navigator.serviceWorker` are NOT available in the SharedWorker, so subscription happens in the page; only the two backend calls go through the Core Worker's authed `RestClient`).

**Backend contract (verified):**
- `GET /push/vapid_public_key` · auth → `{ "public_key": "<base64 VAPID public key>" }`.
- `POST /push/subscribe` · auth → body `{ endpoint, p256dh, auth }` → `{ "ok": true }` (400 if a field is missing). Subscription is keyed by the session **device**.
- Push payload delivered to the SW: `{ chat_id, msg_id, seq, sender: { name }, text, badge }`.
- Push is enqueued by the backend only when the recipient has **no active WS** and the chat is **not muted**. VAPID env: `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`.

**Verification reality (read before planning the verify task):** Real Web Push delivery is **not verifiable in headless Playwright** — `PushManager.subscribe` needs a browser push service and `Notification` permission that headless Chromium doesn't provide. So live verification covers: SW **registers** (`navigator.serviceWorker` ready), `GET /push/vapid_public_key` returns a key once VAPID is set, and the backend **subscribe + enqueue** path works (POST a subscription via API → 200; send a message to an offline subscribed recipient → a job appears on the Redis `push:queue` stream). The actual notification render and end-to-end delivery are documented as not headless-verifiable.

**Tech Stack:** React 18 + TS + Vite (frontend); vanilla JS Service Worker; Go backend (unchanged). The SW lives in `public/sw.js` (Vite copies `public/*` to the build root → served at `/sw.js`, scope `/`).

**Repo topology:** Frontend tasks run in `telegram-ui-clone/` (own git repo). VAPID env goes in the gitignored `docker-compose.verify.yml` at the backend repo root. Frontend commits use the `messenger-denis`/`d.maramygin@…` identity.

```ts
// src/core/managers/pushManager.ts
import type { RestClient } from '../net/restClient'

export interface PushSub { endpoint: string; p256dh: string; auth: string }

export function newPushManager({ rest }: { rest: Pick<RestClient, 'get' | 'post'> }) {
  return {
    async vapidKey(): Promise<string> {
      const r = await rest.get<{ public_key: string }>('/push/vapid_public_key')
      return r.public_key
    },
    async subscribe(sub: PushSub): Promise<{ ok: boolean }> {
      return rest.post<{ ok: boolean }>('/push/subscribe', sub)
    },
  }
}

export type PushManager = ReturnType<typeof newPushManager>
```

## Task 2: Service Worker (push + notificationclick)

**Files:** Create `telegram-ui-clone/public/sw.js`.

**Context:** Plain vanilla JS (no bundling/imports). Shows a notification from the payload; clicking focuses an existing window (and posts `open-chat`) or opens the app. The backend already gates on offline+unmuted, so the SW shows unconditionally.

- [ ] **Step 1: Create `public/sw.js`**

```js
/* Web Push service worker. Scope: / (served from the build root). */
self.addEventListener('install', () => self.skipWaiting())
self.addEventListener('activate', (e) => e.waitUntil(self.clients.claim()))

self.addEventListener('push', (event) => {
  let d = {}
  try { d = event.data ? event.data.json() : {} } catch (_e) { d = {} }
  const title = (d.sender && d.sender.name) || 'New message'
  const body = d.text || ''
  const chatId = d.chat_id
  event.waitUntil(
    self.registration.showNotification(title, {
      body,
      tag: chatId != null ? 'chat-' + chatId : undefined,
      renotify: true,
      data: { chatId },
    }),
  )
})

self.addEventListener('notificationclick', (event) => {
  event.notification.close()
  const chatId = event.notification.data && event.notification.data.chatId
  event.waitUntil(
    self.clients.matchAll({ type: 'window', includeUncontrolled: true }).then((cs) => {
      for (const c of cs) {
        if ('focus' in c) { c.focus(); c.postMessage({ type: 'open-chat', chatId }); return }
      }
      if (self.clients.openWindow) return self.clients.openWindow('/')
    }),
  )
})
```

```ts
import { startClient } from './bootstrap'

function urlBase64ToUint8Array(base64: string): Uint8Array {
  const padding = '='.repeat((4 - (base64.length % 4)) % 4)
  const b64 = (base64 + padding).replace(/-/g, '+').replace(/_/g, '/')
  const raw = atob(b64)
  const out = new Uint8Array(raw.length)
  for (let i = 0; i < raw.length; i++) out[i] = raw.charCodeAt(i)
  return out
}

function keyB64(sub: PushSubscription, name: 'p256dh' | 'auth'): string {
  const buf = sub.getKey(name)
  if (!buf) return ''
  let s = ''
  const bytes = new Uint8Array(buf)
  for (let i = 0; i < bytes.length; i++) s += String.fromCharCode(bytes[i])
  return btoa(s)
}

let done = false

// Register the browser push subscription with the backend (best-effort, idempotent).
export async function setupPush(): Promise<void> {
  if (done) return
  done = true
  try {
    if (!('serviceWorker' in navigator) || !('PushManager' in window) || !('Notification' in window)) return
    const reg = await navigator.serviceWorker.ready
    const perm = Notification.permission === 'default' ? await Notification.requestPermission() : Notification.permission
    if (perm !== 'granted') return
    const { managers } = startClient()
    const vapid = await managers.push.vapidKey()
    if (!vapid) return
    let sub = await reg.pushManager.getSubscription()
    if (!sub) {
      sub = await reg.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(vapid) })
    }
    await managers.push.subscribe({ endpoint: sub.endpoint, p256dh: keyB64(sub, 'p256dh'), auth: keyB64(sub, 'auth') })
  } catch {
    done = false // allow a later retry (e.g. permission granted afterwards)
  }
}
```
